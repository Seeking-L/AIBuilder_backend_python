# 前端修改说明：实时展示 AI 工作过程（异步 + WebSocket）

本说明基于后端已实现的接口：

- 异步启动任务：`POST /tasks/generate-app-async`
- 实时进度推送：`WS /tasks/ws/{taskId}`

下面以 React + fetch + 原生 WebSocket 为例说明。

---

## 一、接口类型定义（前端）

```ts
type AgentEventType =
  | "round_start"
  | "llm_response"
  | "tool_call"
  | "tool_result"
  | "finished"
  | "command_start"
  | "command_output"
  | "command_end";

export interface AgentEvent {
  stepId: number;
  type: AgentEventType;
  title: string;
  detail?: string | null;
}

export interface StartGenerateAppResponse {
  status: "accepted";
  taskId: string;
  expoRoot: string;
}

export interface TaskStatusMessage {
  type: "task_status";
  status: "completed" | "failed";
  error?: string | null;
}
```

---

## 二、命令执行实时输出（新事件类型）

当 AI 执行 shell 命令时，后端会按顺序推送以下事件，便于前端做「终端」式展示。

### 1. `command_start` — 命令开始

- **含义**：即将执行一条命令，用户可立即看到命令内容。
- **示例 payload**：
  ```json
  { "stepId": 5, "type": "command_start", "title": "执行命令", "detail": "npm ci" }
  ```
- **前端建议**：显示一行 `$ npm ci`（或 `> 执行命令: npm ci`）。

### 2. `command_output` — 实时输出块

- **含义**：命令的 stdout/stderr 的一小段输出（按行或按块推送）。
- **示例 payload**：
  ```json
  { "stepId": 5, "type": "command_output", "title": "stdout", "detail": "added 412 packages\n" }
  ```
  ```json
  { "stepId": 5, "type": "command_output", "title": "stderr", "detail": "npm WARN ...\n" }
  ```
- **前端建议**：将同一 `stepId` 下的多条 `command_output` 追加到同一块终端区域；可用 `title === "stderr"` 用不同样式（如红色）区分 stderr。

### 3. `command_end` — 命令结束

- **含义**：命令已结束，可展示退出码或简短总结。
- **示例 payload**：
  ```json
  { "stepId": 5, "type": "command_end", "title": "命令结束", "detail": "Exit code: 0" }
  ```
- **前端建议**：在该命令块末尾显示「命令结束 (Exit code: 0)」或仅显示退出码。

### 4. 终端块 UI 示例

将同一 `stepId` 的 `command_start` → 多条 `command_output` → `command_end` 归为一组，渲染为一个「终端块」：

```tsx
function CommandBlock({ events }: { events: AgentEvent[] }) {
  const start = events.find((e) => e.type === "command_start");
  const outputs = events.filter((e) => e.type === "command_output");
  const end = events.find((e) => e.type === "command_end");

  return (
    <div className="command-block">
      {start?.detail && (
        <div className="command-line">$ {start.detail}</div>
      )}
      <pre className="command-output">
        {outputs.map((e) => (
          <span
            key={e.stepId + e.title + (e.detail ?? "")}
            className={e.title === "stderr" ? "stderr" : "stdout"}
          >
            {e.detail}
          </span>
        ))}
      </pre>
      {end?.detail && (
        <div className="command-end">{end.detail}</div>
      )}
    </div>
  );
}
```

在时间线中，可按 `stepId` 分组：同一 `stepId` 下若存在 `command_start`，则整组用 `CommandBlock` 渲染；否则用原有事件项渲染。

## 三、调用异步启动接口

```ts
async function startGenerateApp(payload: { description: string; framework?: string }) {
  const res = await fetch("/tasks/generate-app-async", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload),
  });

  if (!res.ok) {
    const data = await res.json().catch(() => ({}));
    throw new Error(data.error || "generate-app-async failed");
  }

  const data: StartGenerateAppResponse = await res.json();
  return data;
}
```

流程建议：用户点击「生成应用」→ 调用 `startGenerateApp` 拿到 `taskId`、`expoRoot` → 立即连接 WebSocket `/tasks/ws/${taskId}` → 在 WebSocket 消息里更新 `events` 与任务状态。

## 四、WebSocket 实时订阅任务进度

### 1. 状态设计

```ts
const [taskId, setTaskId] = useState<string | null>(null);
const [events, setEvents] = useState<AgentEvent[]>([]);
const [taskStatus, setTaskStatus] = useState<"idle" | "running" | "completed" | "failed">("idle");
const [error, setError] = useState<string | null>(null);
```

### 2. 建立连接并处理消息

```ts
ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  if (data.type === "task_status") {
    const msg = data as TaskStatusMessage;
    setTaskStatus(msg.status === "completed" ? "completed" : "failed");
    if (msg.error) setError(msg.error);
    ws.close();
    return;
  }

  const evt = data as AgentEvent;
  setEvents((prev) => [...prev, evt]);
};
```

收到 `command_start` / `command_output` / `command_end` 时统一 append 到 `events`，再在 UI 中按 `stepId` 分组并用 `CommandBlock` 展示命令块。

---

## 五、时间线与命令块结合

- 对 `events` 按 `stepId` 分组。
- 若某组内存在 `type === "command_start"`，则用「终端块」组件渲染该组（显示 `$ 命令` + 实时输出 + 结束信息）。
- 其余事件（如 `round_start`、`llm_response`、`tool_call`、`tool_result`、`finished`）仍按原时间线条目展示。

## 六、同步接口兼容
